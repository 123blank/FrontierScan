## 2. 新增接口与类

### 2.1 Collector — 采集器策略接口

```java
package com.frontierscan.collection;

/**
 * 采集器策略接口。
 * <p>
 * 每种内容源类型实现此接口，通过 {@link #sourceType()} 标识来源类型。
 * 当前内置 RSS/Atom 和 HTML 网页两种实现，后续可扩展如 API 采集器等。
 * 使用策略模式，由 {@link CollectionOrchestrator} 根据 Site 配置动态选择。
 * </p>
 */
public interface Collector {

    /** 返回采集器类型标识（如 "RSS"、"HTML"）。 */
    String sourceType();

    /**
     * 执行采集。
     *
     * @param site 信息源网站配置（含 URL、RSS 地址等）
     * @return 采集结果，包含原始文章列表和采集元信息
     * @throws CollectorException 采集过程中发生的可恢复异常
     */
    CollectResult collect(Site site) throws CollectorException;
}
```

### 2.2 CollectResult — 采集结果封装

```java
package com.frontierscan.collection;

import java.time.Instant;
import java.util.List;
import java.util.Objects;

/**
 * 采集器执行结果。
 *
 * @param sourceType    采集器类型（RSS / HTML）
 * @param rawArticles   原始文章列表（尚未去重和摘要）
 * @param collectedAt   采集时间
 * @param fetchDuration 抓取耗时（毫秒）
 * @param parseCount    解析出的文章数
 * @param errorMessage  采集错误信息（成功时为 null）
 */
public record CollectResult(
        String sourceType,
        List<RawArticle> rawArticles,
        Instant collectedAt,
        long fetchDuration,
        int parseCount,
        String errorMessage
) {

    /**
     * 原始文章数据。
     * <p>
     * 采集器输出 → 去重/摘要/落库的中间格式。
     * sourceHash 若未传入，根据 sourceUrl 自动生成。
     * </p>
     */
    public record RawArticle(
            String title,
            String sourceUrl,
            String content,
            String contentExcerpt,
            Instant publishedAt,
            String sourceHash
    ) {
        public RawArticle {
            Objects.requireNonNull(title, "文章标题不能为空");
            Objects.requireNonNull(sourceUrl, "原文链接不能为空");
            if (sourceHash == null || sourceHash.isBlank()) {
                sourceHash = ArticleParser.generateSourceHash(sourceUrl);
            }
        }
    }
}
```

### 2.3 CollectorException — 异常层次

```java
package com.frontierscan.collection;

/**
 * 采集过程基类异常。
 * 所有 Collector 抛出的异常继承此类，确保错误信息结构化。
 */
public class CollectorException extends RuntimeException {
    private final String sourceType;
    private final String siteUrl;
    private final String errorCode;
}

/** 连接超时异常：网站不可达或超时。致命异常，标记任务 FAILED。 */
public class ConnectionTimeoutException extends CollectorException {}

/** 解析异常：RSS/HTML 格式异常。可降级异常（如 RSS 失败 → HTML）。 */
public class ParseException extends CollectorException {}

/** 空结果异常：已是最新，无新内容。标记 COMPLETED(collectedCount=0)。 */
public class EmptyResultException extends CollectorException {}
```

## 3. 采集器实现

### 3.1 RssCollector — RSS/Atom 采集

```
核心逻辑：
1. 使用 Rome 库解析 RSS/Atom Feed
2. 提取每条文章的 title / link / description / publishedDate
3. 调用 ArticleParser 提取 HTML 正文（RSS 摘要通常不完整）
4. 组装 CollectResult 返回

设计要点：
- Rome 自动识别 RSS 0.91/0.92/1.0/2.0 和 Atom 格式
- 连接超时 15s，读取超时 30s
- 单次采集上限 50 条
- 记录抓取耗时和条目解析数量
```

### 3.2 HtmlCollector — 网页抓取采集

```
核心逻辑：
1. 使用 Jsoup 抓取目标网页 HTML
2. 通用规则提取文章链接（查找 <article> / <a> 等模式）
3. 逐个抓取文章详情页
4. 调用 ArticleParser 提取正文
5. 组装 CollectResult 返回

设计要点：
- Jsoup 配置 User-Agent 模拟浏览器
- 连接超时 15s，读取超时 30s
- 单次采集上限 20 条（网页解析比 RSS 重）
- 配置 referrer 和 cookie 处理基础反爬
- 只抓取同域名内容，防止链接污染
```

## 7. 容错与降级策略

| 场景 | 处理方式 |
|---|---|
| RSS 解析失败 | 自动降级到 HTML 采集器 |
| HTML 抓取超时 | 记录错误，标记任务 FAILED，不重试 |
| 部分文章去重成功 | 只保存新文章，不中断整个任务 |
| 空结果（无新文章） | 标记为 COMPLETED，collectedCount=0 |
| 网络中断异常 | 标记 FAILED，保留 errorMessage 供后续诊断 |
| 文章正文提取失败 | 保存 title + sourceUrl，summary/content 留空 |
