# 个人用于进行新增业务的说明提示词，Agent不需要自行读取

接下来我需要完成以下新的业务

业务描述：

要求：

1. 不要直接修改代码，优先制定方案
2. 优先查看本地代码，是否有类似功能，保持业务风格一致
3. 优先查看目前启动的浏览器页面，了解相关业务功能的情况
4. 给出需要新增业务的现状描述以及需要影响的
5. 修改代码是保持最小修改原则，不要影响到无关的业务
6. 采用TTD模式进行业务开发，测试驱动开发，确保业务正确性
7. 业务一定要进行优化，减少上线后出现崩溃宕机的风险
8. 对于不确定的地方不要自行决定，需要与我进行协商确认
9. 将一个业务开发完成测试优化好后，再进行下一个业务的开发
10. 业务方案计划文档需要保存，开发完成后需要总结成一个报告文档

测试流程：

先确认你刚更新后的工作区差异和分支，再跑后端窄测试、Docker 重新构建启动、浏览器自动化验收，最后关闭 Docker。
