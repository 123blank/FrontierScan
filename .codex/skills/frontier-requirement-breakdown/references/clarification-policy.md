# Clarification Policy

Ask questions only when the answer blocks safe planning.

## Ask Immediately When

- The target user/workflow is unknown.
- The requested behavior conflicts with existing project conventions.
- Data deletion, migration, publishing, auth, or security behavior is unclear.
- Acceptance criteria cannot be written without guessing.
- The request depends on an unavailable external system or credential.

## Do Not Ask When

- The missing detail can be discovered from `llm-knowledge/` or source code.
- A conservative default is obvious and low risk.
- The detail can be recorded as a non-blocking assumption.

## Output Rules

- Separate assumptions from blocking questions.
- Keep questions short and numbered.
- If there are no blockers, proceed with assumptions and record them.
