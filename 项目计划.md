# FrontierScan Agent 项目开发计划

## Summary
构建一个企业级 Web Agent 系统，用于采集、整理和展示技术/AI 前沿网站信息。用户可自定义分类和网站，系统支持手动采集与定时采集，优先通过 RSS/Atom 获取内容，缺失时使用网页解析。采集结果由阿里大模型进行摘要、标签和要点整理，并按分类以摘要卡片优先展示，点击后查看详情和原文地址。

技术栈采用 Java + Vue：后端 Spring Boot 3，前端 Vue 3 + TypeScript + Vite，单机 Docker Compose 部署，保留后续拆分和多模型扩展能力。

## Key Changes
- 项目初始化为前后端分离架构：
  - `backend`: Spring Boot 3 REST API 服务。
  - `frontend`: Vue 3 管理与阅读界面。
  - `deploy`: Docker Compose、Nginx、环境变量模板。
- 后端核心模块：
  - 用户与基础权限：管理员、普通用户，JWT 登录，用户数据隔离。
  - 分类管理：用户可创建、编辑、排序、归档分类。
  - 网站管理：添加网站名称、URL、分类、RSS 地址、采集频率、启停状态。
  - 采集任务：支持手动触发与定时调度，记录执行状态、耗时、错误信息。
  - 内容处理：文章抓取、正文抽取、去重、摘要生成、标签提取、原文链接保留。
  - 阅读辅助：搜索、收藏、按分类/来源/标签/时间筛选。
  - 基础可观测：健康检查、任务日志、失败重试、结构化日志。
- 大模型设计：
  - 默认实现阿里 DashScope/Qwen Provider。
  - 设计 `LlmProvider` 抽象，配置支持 `provider`, `apiKey`, `baseUrl`, `model`。
  - 摘要输出统一结构：标题优化、三到五句摘要、关键要点、标签、可信原文 URL。
- 数据存储：
  - PostgreSQL 作为主数据库。
  - Redis 用于任务锁、限流和可选缓存。
  - Flyway 管理数据库迁移。
- 前端体验：
  - 首页为分类化信息看板，不做营销页。
  - 左侧分类导航，主区域展示摘要卡片，顶部支持搜索和筛选。
  - 详情以抽屉或详情页展示：摘要、要点、标签、来源、采集时间、原文地址。
  - 管理区包含分类管理、网站管理、采集任务记录、个人收藏。
  - UI 风格保持企业级、克制、清晰，优先扫描效率和长期使用舒适度。

## Public Interfaces
- 后端 REST API：
  - `POST /api/auth/login`
  - `GET/POST/PUT/DELETE /api/categories`
  - `GET/POST/PUT/DELETE /api/sites`
  - `POST /api/sites/{id}/collect`
  - `GET /api/articles`
  - `GET /api/articles/{id}`
  - `POST/DELETE /api/articles/{id}/favorite`
  - `GET /api/collection-runs`
  - `GET /actuator/health`
- 关键数据模型：
  - `User`: 账号、密码哈希、角色、状态。
  - `Category`: 名称、描述、排序、所属用户。
  - `Site`: 名称、URL、RSS URL、分类、采集频率、启停状态。
  - `Article`: 标题、摘要、要点、标签、正文片段、原文 URL、发布时间、来源网站、分类。
  - `CollectionRun`: 任务类型、状态、开始/结束时间、错误信息、采集数量。
  - `Favorite`: 用户与文章收藏关系。

## Test Plan
- 后端：
  - 单元测试覆盖分类、网站、去重、摘要 Provider 选择、任务状态流转。
  - 集成测试覆盖登录、网站新增、手动采集、文章查询、收藏接口。
  - 使用 mock LLM Provider，避免测试依赖真实大模型。
- 前端：
  - 组件测试覆盖分类导航、摘要卡片、详情展示、网站表单。
  - E2E 覆盖登录、添加分类、添加网站、触发采集、查看详情、收藏文章。
- 部署验证：
  - `docker compose up` 后可访问前端。
  - 后端健康检查正常。
  - 数据库迁移自动执行。
  - 未配置大模型 Key 时，系统明确提示摘要不可用，但采集和展示不崩溃。

## Assumptions
- 第一版聚焦技术/AI 前沿网站，不做多行业资讯模板。
- 分类由用户手动维护为主，AI 仅辅助生成标签和摘要。
- 采集优先 RSS/Atom，网页解析作为补充，不做复杂反爬绕过。
- 第一版采用基础账号权限，不实现完整 RBAC。
- 部署目标为单机 Docker Compose，但代码按模块化方式组织，后续可演进为分布式任务或微服务。
